Three Trust Levels

Choose the security level your use case requires. More trust = more blockchain.

Level 1

APS Basic

Format + signatures. No blockchain required.

  • Standardized JSON format
  • Ed25519 digital signatures
  • Keccak-256 hash chains
  • Local verification
  • No verifiable timestamps
  • No tamper detection
Level 2

APS Anchored

Immutability + verifiable timestamps.

  • Everything in Basic
  • Blockchain proof-of-existence
  • Verifiable timestamps (block #)
  • Tamper detection
  • Batch Merkle anchoring
  • No heritage/attestation
Level 3

APS Full

Complete security. On-chain identity.

  • Everything in Anchored
  • Heritage & lineage DAG
  • Attestation exchange
  • On-chain governance
  • Agent Identity Registry
  • Memory vault integrity

Cryptographic Foundation

Every APS artifact is built on battle-tested cryptographic primitives:

  • Ed25519 ‚Äî Digital signatures (RFC 8032). Fast, small keys, deterministic.
  • Keccak-256 ‚Äî Hashing (FIPS 202). EVM-compatible, collision-resistant.
  • RFC 8785 ‚Äî JSON Canonicalization. Deterministic serialization across languages.
  • Merkle Trees ‚Äî Batch proof. Anchor 1000 hashes for the cost of 1 transaction.
  • did:key ‚Äî Decentralized Identifiers. Self-sovereign, no registry needed.
